## Context
- This is a nix-darwin configuration repository at `~/.config/nix`
- Builds are run via `make build HOST=<hostname> FLAGS=switch`
- The build output includes homebrew bundle, home-manager activation, and nix profile steps
- Common issues: duplicate app installs (mas vs brew casks), missing generated files, Swift/SDK version mismatches, failed brew fetches

## Your Task

Diagnose issues from a nix-darwin build output pasted by the user.

### Steps

1. Parse the pasted build output and identify anomalies:
   - Apps listed as "Installing" that should show "Using" (indicates re-installation)
   - Error messages from brew, mas, home-manager, or nix
   - Swift/SDK version mismatch errors during brew cask operations
   - Missing activation steps or failed plugin installations
2. Read the relevant `.nix` configuration files to understand root cause:
   - `hosts/<hostname>/homebrew.nix` for brew/cask declarations
   - `hosts/<hostname>/mas.nix` for Mac App Store declarations
   - `modules/` for shared module definitions
   - Check for duplicate package declarations across brew casks and mas
3. Explain the root cause clearly and propose a minimal fix
4. If the fix involves moving packages between mas and brew (or vice versa), show the exact changes needed in both files

### Guardrails
- Microsoft Office apps should be installed via brew casks, NOT mas (learned from past corrections)
- Do not generate `known_marketplaces.json` from nix — it causes drift risk and file ownership conflicts
- Always check if a package appears in BOTH mas and homebrew declarations before diagnosing duplicates
- When brew cask upgrades fail with Swift errors, this is typically a macOS/Xcode Command Line Tools version mismatch — not a nix issue
