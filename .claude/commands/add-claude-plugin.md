## Context
- Claude Code plugins are managed declaratively via `claude.nix` in this nix-darwin repo
- Plugins are installed imperatively via activation scripts (`claude plugin marketplace add` + `claude plugin install`)
- The `known_marketplaces.json` file is managed by the Claude CLI, NOT by nix
- Skills from GitHub repos are installed via `claude mcp add` or local `.claude/commands/` files

## Your Task

Add a new Claude Code plugin or skill to the nix-darwin configuration so it is installed on every rebuild.

### Steps

1. Determine what type of addition this is:
   - **Marketplace plugin** (e.g., `bayramannakov/claude-reflect`): needs marketplace add + plugin install
   - **GitHub skill** (e.g., `https://github.com/coleam00/excalidraw-diagram-skill`): needs skill installation logic
   - **Local skill file** (`.claude/commands/*.md`): just needs the file created
2. Read the current `claude.nix` (or `modules/claude.nix` / `hosts/<hostname>/claude.nix`) to understand the existing structure
3. Add the plugin/skill declaration following the existing pattern in `claude.nix`
4. Ensure the activation script handles the installation correctly

### Guardrails
- Do NOT generate `known_marketplaces.json` from nix — drift risk and file ownership conflicts with CLI
- Do NOT generate `installed_plugins.json` from nix — the CLI manages versions, commit SHAs, and install paths
- Use imperative `claude plugin install` commands in activation scripts instead
- The activation script should be idempotent (check if already installed before re-installing)
- Marketplace cloning may fail over HTTPS in some environments; the CLI handles SSH fallback automatically
