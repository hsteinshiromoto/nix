## Context
- This nix-darwin repo uses a "dendritic" design pattern for host-specific configuration
- Dendritic pattern means: a module accepts the hostname as a parameter and conditionally applies different settings per host
- Host directories are under `hosts/<hostname>/` (e.g., `hosts/mbp2025/`, `hosts/mbp2023/`)
- Shared modules are under `modules/`

## Your Task

Refactor a `.nix` module file to use the dendritic (host-branching) pattern.

### Steps

1. Read the target `.nix` file to understand its current structure
2. Identify which settings should be host-specific vs shared
3. Read existing dendritic modules (e.g., `mcp.nix`, `gitlab.nix`) for reference patterns
4. Refactor the module to:
   - Accept hostname as a parameter (typically via `config.networking.hostName` or function argument)
   - Use `lib.mkIf` or `if-then-else` to branch settings by host
   - Keep shared/common settings outside the conditional blocks
5. Verify that all existing hosts still receive their expected configuration

### Guardrails
- Always check what hosts exist under `hosts/` before refactoring
- Preserve backward compatibility — existing host behavior should not change
- When moving secret references, update the SOPS path to include the hostname (e.g., `~/.config/sops/secrets/<hostname>/`)
- If the user specifies SSH keys vs API tokens, use the correct authentication method per host
- Use `lib.mkMerge` for combining conditional and unconditional settings cleanly
